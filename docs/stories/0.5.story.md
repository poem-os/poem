# Story 0.5: Rename POEM Core Workflows Directory to Tasks for BMAD Alignment

## Status

Done

## Story

**As a** POEM developer and user,
**I want** the POEM core agent procedure directory renamed from `workflows/` to `tasks/` to match BMAD terminology,
**so that** the codebase uses consistent terminology (agents execute "tasks", not "workflows") and eliminates confusion between agent procedures and user workspace projects.

## Background

**Terminology Collision**: POEM currently uses "workflow" for two distinct concepts:
1. **Agent Procedures** (packages/poem-core/workflows/) - Step-by-step SOPs that agents execute (e.g., new-prompt.yaml, refine-prompt.yaml)
2. **User Workspace Projects** (dev-workspace/workflows/<project-name>/) - Collections of prompts for specific domains (YouTube, NanoBanana)

**BMAD Precedent**: The BMAD Method framework uses clear terminology:
- **Agents** = Personas (Bob, James, Quinn)
- **Tasks** = Step-by-step procedures agents execute (.bmad-core/tasks/)
- **Commands** = User-invocable operations (*draft, *review)

**Problem**: Documentation frequently needs clarification like "agent workflows" vs "user workflows" or "workflow definitions" vs "workflow projects", creating cognitive overhead.

**Solution**: Rename `packages/poem-core/workflows/` → `packages/poem-core/tasks/` to align with BMAD terminology and eliminate ambiguity.

**Scope**: This story ONLY affects POEM core agent procedure files. It does NOT touch:
- `dev-workspace/workflows/` (user workspace projects - stays as-is)
- `.bmad-core/workflows/` (BMAD greenfield/brownfield patterns - different context)
- Multi-workflow architecture (Story 3.8 - separate concept)

**References**:
- Triage Decision: Epic 0 (maintenance/refactor, 1-2 hour effort)
- BMAD Structure: `.bmad-core/tasks/` contains agent procedures
- POEM Agent: `packages/poem-core/agents/prompt-engineer.md` (line 15-16, 57-60)
- Conversation Context: User requested alignment with BMAD terminology

## Dev Notes

### BMAD Task Structure (Reference)

**Source**: `.bmad-core/` directory structure

BMAD organizes agent procedures as "tasks":
```
.bmad-core/
├── agents/              # Agent personas (Bob, James, Quinn)
├── tasks/              # ← Agent procedures (create-next-story.md, review-story.md)
├── workflows/          # High-level patterns (greenfield/brownfield)
├── templates/          # Output templates
└── checklists/         # Validation checklists
```

**Terminology**:
- **Agent** = The "who" (Bob the Scrum Master)
- **Command** = The "what" user types (*draft, *review)
- **Task** = The "how" - step-by-step SOP (create-next-story.md)
- **Workflow** = High-level pattern (greenfield vs brownfield)

**Example Flow**:
```
User: /BMad/agents/sm
Bob: [activates] *help shows commands

User: *draft
Bob: [executes TASK: create-next-story.md]
     Step 0: Load core-config.yaml
     Step 1: Identify next story
     Step 2: Gather requirements
     ...
```

### The Naming Collision

**Problem**: POEM uses "workflow" for TWO concepts:

1. **Agent Procedures** (packages/poem-core/workflows/) - What BMAD calls "tasks"
   - Step-by-step SOPs agents execute
   - Examples: new-prompt.yaml, refine-prompt.yaml

2. **User Workspace Projects** (dev-workspace/workflows/<name>/) - Different concept entirely
   - Collections of prompts for specific domains
   - Examples: youtube-launch-optimizer/, nano-banana/
   - Implemented in Story 3.8 (Multi-Workflow Foundation)

**Confusion Examples in Codebase**:
- "workflow definitions" (means agent procedures)
- "multi-workflow architecture" (means user workspace projects)
- "workflow data" (which workflow??)
- "workflow context" (ambiguous!)

### Target Structure (After Rename)

```

├── agents/              # Agent personas
├── tasks/              # ← RENAMED (agent procedures)

dev-workspace/
└── workflows/          # ← UNCHANGED (user workspace projects)

```

**New Terminology**:
- **Agents** = Personas (Penny, Felix, Victor)
- **Tasks** = Step-by-step procedures (new-prompt, refine-prompt)
- **Skills** = Helper operations
- **Projects** = User workspace collections (YouTube, NanoBanana) - currently called "workflows"

Note: Story 3.8 renamed to "projects" is out of scope for THIS story. This story only addresses the agent procedure directory.
