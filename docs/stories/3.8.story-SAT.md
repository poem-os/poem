## Troubleshooting

### Issue: Server won't start

**Symptom**: `npm run dev` fails with port error
**Solution**:
- Check if port 4321 is in use: `lsof -i :4321`
- Kill existing process or use different port

### Issue: Penny doesn't show new commands

**Symptom**: `*help` doesn't list `*workflows`, `*switch`, `*context`
**Solution**:
- Verify you're in correct directory: `packages/poem-app/`
- Check agent file updated: `grep "*workflows" packages/poem-core/agents/prompt-engineer.md`
- Try reactivating Penny: `/poem/agents/penny`

### Issue: Workflows not found

**Symptom**: `*workflows` shows "No workflows defined"
**Solution**:
- Verify config file has uncommented workflows: `grep "currentWorkflow:" packages/poem-core/poem-core-config.yaml`
- Check POEM_DEV environment variable set: `cat .env | grep POEM_DEV`
- Restart dev server if config was just updated

### Issue: Prompts not isolated

**Symptom**: Prompts from workflow A appear in workflow B
**Solution**:
- Verify current workflow: `*context`
- Check prompt file location matches workflow directory
- Verify workflow switching worked: `*workflows` (check for ‚úì ACTIVE indicator)

### Issue: Persistence not working

**Symptom**: Workflow resets after Penny restart
**Solution**:
- Check persistence file exists: `ls dev-workspace/.poem-state.json`
- Verify file contents: `cat dev-workspace/.poem-state.json`
- Check file permissions (should be writable)
